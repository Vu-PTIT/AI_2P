# AI Pipeline - Real-Time VI-EN Meeting Translator

This folder contains the AI worker used by `realtime-service`.

The worker now expects real AI backends:

- ASR: `openai-whisper`
- Audio: numpy noise gate or optional `noisereduce`, Silero VAD through
  `torch.hub`, channel-based speaker hints, optional pyannote diarization
- Fast translation: NLLB/seq2seq through HuggingFace `transformers`
- Quality translation: FPT AI Factory or any OpenAI-compatible chat API
- RAG: `sentence-transformers` embeddings plus local Qdrant, with TXT/MD/JSON,
  PDF, DOCX, and PPTX ingest

No sample transcript or hard-coded translation data is generated by the worker.
If a required model or API is missing, the worker emits an `error` event instead
of inventing text.

## Run

Install dependencies and make sure the model weights/API endpoint are available:

```bash
cd ai
pip install -r requirements.txt
copy .env.example .env
python main.py
```

Point the NestJS service at it:

```bash
AI_WS_URL=ws://127.0.0.1:8765/ws/session
```

Useful checks:

```bash
python main.py --check
python -m tests.test_smoke
```

Ingest meeting documents into RAG before a meeting:

```bash
python main.py --ingest-doc ./data/agenda.txt
```

## Environment

```bash
AI_WS_HOST=127.0.0.1
AI_WS_PORT=8765

WHISPER_MODEL=large-v3
WHISPER_MODEL_DIR=./models/whisper
WHISPER_DEVICE=cpu

AUDIO_DENOISE=gate
AUDIO_DIARIZATION=off
PYANNOTE_MODEL=pyannote/speaker-diarization-3.1
# PYANNOTE_AUTH_TOKEN=replace-with-your-huggingface-token

FAST_MT_MODEL=facebook/nllb-200-distilled-600M
FAST_MT_DEVICE=cpu

FPT_AI_FACTORY_BASE_URL=https://your-fpt-ai-factory-endpoint/v1
FPT_AI_FACTORY_API_KEY=replace-with-your-key
FPT_AI_FACTORY_MODEL=your-fpt-model
FPT_AI_FACTORY_TIMEOUT=1.0

# Optional non-FPT aliases:
# QUALITY_LLM_BASE_URL=http://localhost:8000/v1
# QUALITY_LLM_API_KEY=local
# QUALITY_LLM_MODEL=your-llm-model
# QUALITY_LLM_TIMEOUT=1.0

RAG_EMBEDDING_MODEL=sentence-transformers/all-MiniLM-L6-v2
RAG_QDRANT_PATH=./data/qdrant_local
RAG_COLLECTION=meeting_docs
RAG_ALLOWED_ROOT=./data

SESSION_EXPORT_DIR=./data/session_exports
OVERLAP_BUFFER_LIMIT=20
```

At runtime, the worker also accepts a WebSocket control message:

```json
{"type":"rag.ingest","paths":["./data/agenda.txt"]}
```

`RAG_ALLOWED_ROOT` limits which local files the worker may ingest.

## WebSocket Events

The existing event contract is still present: `stt.partial`, `stt.final`,
`translate.token`, `translate.done`, and `error`.

Additional completion events:

```json
{"type":"stt.revision","utteranceId":"...","oldText":"...","newText":"...","semanticChange":true,"diff":[]}
{"type":"audio.overlap","utteranceId":"...","strategy":"dominant-speaker-with-buffer","bufferedCount":1}
{"type":"system.status","level":1,"status":"reduce_context","networkOk":true,"gpuAvailable":true,"micOk":true}
{"type":"rag.ingested","chunks":12}
```

Set `AUDIO_DIARIZATION=pyannote` plus `PYANNOTE_AUTH_TOKEN` to use pyannote on
shared-mic audio. For separate physical mics, send separate clients/channels;
the worker assigns the dominant channel as `speaker_1`, `speaker_2`, and so on.

## Structure

```text
config/       deployment, glossary, acronym loading
rag/          SentenceTransformers + Qdrant retrieval
audio/        PCM normalization + Silero VAD
asr/          Whisper ASR plus revision handler
translation/  NLLB fast path + FPT/OpenAI-compatible quality path
fallback/     health monitor and degradation levels
session/      sliding window, minutes, export
worker.py     WebSocket worker contract for realtime-service
main.py       CLI entrypoint
```
